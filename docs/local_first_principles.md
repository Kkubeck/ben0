# Local-First Principles

BEN-0 is guided by ten working principles:

1. **Local first**  
   Core workflows must run on a local machine without cloud dependence.

2. **Data sovereignty**  
   The institution keeps control of its collection records, notes, and sensitive context.

3. **Human authority**  
   BEN-0 advises; curators and registrars decide.

4. **Evidence based**  
   Answers and reports should point back to records, issues, documents, or metrics.

5. **Deterministic validation**  
   Data quality checks should be inspectable, repeatable, and not dependent on model mood.

6. **Model agnostic**  
   The assistant layer should work with a mock adapter, Ollama, or another compatible backend.

7. **Standards aligned**  
   Schema and terminology should stay compatible with botanical collection and biodiversity data practices where practical.

8. **Sensitive data by design**  
   Rare plant locations, donor details, permit context, and culturally sensitive information must be handled conservatively.

9. **Curator trust over automation theater**  
   BEN-0 should be useful because it is careful, not because it appears magical.

10. **Prototype discipline**  
   Keep the stack understandable, portable, and easy to deploy on modest hardware.
